## 코딩 행동 지침 (Claude Code 내장 규칙 보충)

### 구현 전 의사소통
- 여러 해석이 가능하면 조용히 하나를 고르지 말고 선택지를 제시하라
- 더 단순한 방법이 있으면 말하라. 필요할 때 반론을 제기하라
- 불분명하면 멈추고 뭐가 헷갈리는지 말한 뒤 물어라

### 외과적 수정 - 죽은 코드 정책
- 내 변경으로 사용되지 않게 된 import/변수/함수는 즉시 제거
- 기존에 있던 죽은 코드는 요청받지 않는 한 삭제하지 말고 언급만
- 기존 코드 스타일이 내 스타일과 달라도 기존 스타일에 맞춘다

### 변경 추적성
- 변경된 모든 줄은 사용자의 요청으로 직접 추적 가능해야 한다

### 목표 주도 실행
- 작업을 검증 가능한 목표로 변환:
  - "유효성 검사 추가" → "잘못된 입력에 대한 테스트를 쓰고, 통과시켜라"
  - "버그 수정" → "재현하는 테스트를 쓰고, 통과시켜라"
  - "리팩터링" → "전후로 테스트가 통과하는지 확인하라"
- 다단계 작업에는 계획을 명시: `[단계] → 검증: [확인사항]`
